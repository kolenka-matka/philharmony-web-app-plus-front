ГОТОВЫЙ ПРОЕКТ: backend + frontend вместе
=========================================

Здесь уже ВСЁ подключено. Тебе нужно только запустить. Ручных правок не требуется.

Что я изменил по сравнению с твоим бэком:
  1. Добавил фронтенд в  src/main/resources/static/app/  (index.html, styles.css, app.js, events.json).
  2. Добавил контроллер  controllers/EventApiController.java  — отдаёт события по /api/events.
  3. В  config/AppSecurityConfiguration.java  добавил две строки, чтобы /app/** и /api/** были
     доступны без логина.
  4. В app.js фронт берёт данные с  fetch("/api/events").

ПОЧЕМУ РАНЬШЕ НЕ РАБОТАЛО:
  Твой /api/events возвращал JSON в виде ["java.util.ArrayList", [["java.util.LinkedHashMap", {...}]...]]
  — с именами Java-классов внутри. Так получалось из-за того, что в RedisConfig объявлен
  @Primary ObjectMapper с activateDefaultTyping(...), и Spring использовал его не только для
  кэша Redis, но и для ВСЕХ HTTP-ответов. Браузер показывал текст нормально, а фронту прилетал
  массив с класс-обёртками вместо [{...},{...}], календарь на этом падал, и срабатывало
  сообщение про "локальный сервер".
  Контроллер EventApiController теперь собирает ответ обычным ObjectMapper и возвращает чистый
  массив [{...},{...}]. Твою настройку кэша Redis я при этом не трогал — она работает как была.


КАК ЗАПУСТИТЬ
------------
1. Подними Postgres и Redis (как в прошлом семестре):
     PostgreSQL  ->  localhost:5433, база philharmonic-db, пользователь postgres, пароль osdfinale
     Redis       ->  localhost:6379
   Если через Docker:
     docker run -d --name pg -e POSTGRES_PASSWORD=osdfinale -e POSTGRES_DB=philharmonic-db -p 5433:5432 postgres:16
     docker run -d --name redis -p 6379:6379 redis:7

2. Запусти приложение из корня проекта:
     Windows:    mvnw.cmd spring-boot:run
     Mac/Linux:  ./mvnw spring-boot:run
   Дождись запуска на порту 8080.

3. Открой в браузере:
     http://localhost:8080/app/index.html

   Проверить сам эндпоинт (должен быть чистый массив объектов):
     http://localhost:8080/api/events


ЕСЛИ СПИСОК ПУСТОЙ ([])
----------------------
Значит в базе нет мероприятий. Наполни её как обычно (data.sql или админ-форма
"Добавить мероприятие" в приложении), и события появятся на странице.


МЕЛОЧЬ ПРО КАРТИНКУ
------------------
У мероприятия "Вечер классической музыки" в базе старая ссылка на mos.ru, которая
больше не открывается — вместо картинки будет серая заглушка (это нормально, вёрстка
не ломается). Если хочешь живое фото, поменяй imageUrl этого события в базе на рабочую
ссылку.


ЗАПАСНОЙ РЕЖИМ (данные из файла, без базы)
------------------------------------------
Если нужно показать страницу без обращения к БД — в app.js поменяй
  fetch("/api/events")  обратно на  fetch("events.json")
Тогда данные возьмутся из static/app/events.json, который отдаёт тот же сервер.
